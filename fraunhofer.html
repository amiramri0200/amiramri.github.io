Technical Deep Dive: Custom IPC Bridge

One of the core challenges was connecting the OMNeT++ network simulator (C++) with Python Machine Learning libraries (PyTorch) without the high latency of TCP sockets.

To solve this, I engineered a low-level Inter-Process Communication (IPC) bridge using POSIX Shared Memory and Semaphores. This allowed for near-instantaneous state transfer between the C++ simulation environment and the Python training loop.

Core C++ Implementation (Snippet)

Below is a snippet of the custom header file I developed to handle the memory mapping and synchronization:


// Shared Data Structure (resides in RAM, accessible by both C++ and Python)
struct SharedBridgeData {
    int current_observation;    // State observation from Simulator
    int current_action;         // Action decided by RL Agent
    volatile bool obs_ready;    // Sync Flag
    volatile bool action_ready; // Sync Flag
    
    // Process IDs for debugging
    int omnet_pid;             
    int python_pid;            
};

class SharedMemoryBridge {
public:
    SharedMemoryBridge(bool is_creator) {
        // 1. Create/Open POSIX Shared Memory
        // O_CREAT | O_EXCL ensures we control the creation lifecycle
        shm_fd = shm_open(SHM_NAME, O_CREAT | O_RDWR, 0666);
        
        // 2. Map Shared Memory into Process Address Space (Zero-Copy)
        shared_data = static_cast(
            mmap(nullptr, sizeof(SharedBridgeData), 
            PROT_READ | PROT_WRITE, MAP_SHARED, shm_fd, 0)
        );

        // 3. Initialize Semaphores for precise synchronization
        obs_semaphore = sem_open(OBS_SEM_NAME, O_CREAT, 0666, 0);
        action_semaphore = sem_open(ACT_SEM_NAME, O_CREAT, 0666, 0);
        
        std::cout << "[SharedBridge] Initialized (PID: " << getpid() << ")" << std::endl;
    }

    // Example: Method to send observation to Python
    void send_observation(int obs, float rew, bool d) {
        shared_data->current_observation = obs;
        shared_data->obs_ready = true;
        
        // Memory barrier to ensure writes are visible
        __sync_synchronize(); 
        
        // Signal Python process to wake up
        sem_post(obs_semaphore);
    }
};

This implementation utilized mmap for zero-copy data access and sem_wait/sem_post for precise synchronization, ensuring the reinforcement learning training pipeline was not bottlenecked by data transfer speeds.
